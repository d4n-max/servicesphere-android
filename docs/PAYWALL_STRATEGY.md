# ServiceSphere Paywall Strategy

ServiceSphere is a field service management app for solo tradespeople and small service businesses. The paywall should feel like upgrading to run a more organized service business, not like blocking basic work.

The core principle is: do not block the user before they understand the app. Let users create value first, then introduce premium when they hit a business-relevant limit or intentionally request a professional feature.

## 1. Recommended Free Tier

The free tier should let a tradesperson understand the full workflow and create real business value before being asked to pay.

| Area | Recommended Free Limit |
| --- | ---: |
| Clients | 5 active clients |
| Jobs | 10 total jobs or 5 active jobs |
| Quotes | 3 quotes |
| Invoices | 3 invoices |
| Notes | Included |
| Job photos | Limited, for example 10 photos total |
| Signatures | 1 sample signature or unavailable until Pro |
| PDF export | Watermarked preview only, or 1 export |
| Branding | ServiceSphere branding only |
| Reports | Basic dashboard only |

The free tier should answer: "Can this app help me run tomorrow's jobs better?"

## 2. Recommended Premium Tier

Recommended paid tier: **ServiceSphere Pro**.

ServiceSphere Pro should include:

- Unlimited clients
- Unlimited jobs
- Unlimited quotes
- Unlimited invoices
- PDF quote and invoice export
- Invoice export and sharing
- Signature capture
- Job photo history
- Business branding on PDFs
- Advanced reports
- Future cloud backup/sync
- Future team/multi-user features

Positioning:

> Run your service business professionally from your phone.

## 3. Features That Should Stay Free

Keep the core activation workflow free:

- Create clients
- Create jobs
- Add job notes
- Add basic job photos
- Create a small number of quotes
- Create a small number of invoices
- View job history
- Basic dashboard
- Route/address access if already part of the workflow
- Basic field records

Do not paywall the user before they have completed at least one useful workflow.

## 4. Features That Should Be Premium

Premium should map to business scale, professionalism, and recordkeeping:

- More or unlimited clients
- More or unlimited jobs
- Unlimited quotes
- Unlimited invoices
- PDF quote/invoice export
- Invoice export and sharing
- Signature capture
- Full job photo history
- Business branding
- Advanced reports
- Future cloud backup/sync
- Future team access

Best premium logic: free users can run a few real jobs; paid users can run the business.

## 5. Best Paywall Trigger Moments

Use business-relevant triggers:

- User tries to add their 6th client
- User tries to create their 11th job
- User tries to create or export another quote after the free quote limit
- User taps **Export PDF**
- User taps **Add signature**
- User taps **Add logo** or **Business branding**
- User opens advanced reports
- User has completed 2-3 jobs and returns to the app later
- User creates an invoice and taps **Send** or **Export**

The strongest trigger is after the user finishes a quote, invoice, or job. That is when the value is obvious.

## 6. Worst Paywall Trigger Moments To Avoid

Avoid showing the paywall:

- On first launch
- During onboarding
- Before first client/job creation
- While saving job notes
- While capturing a required field record
- When viewing existing records
- When trying to access saved data
- Immediately after install
- Repeatedly after dismissal
- During an urgent on-site workflow

For field service apps, trust matters. Never make the app feel like it is holding work hostage.

## 8. Premium Benefits Hierarchy

Lead with outcomes, not feature inventory.

1. **Keep growing without limits**  
   Unlimited clients, jobs, quotes, and invoices.

2. **Look professional with every customer**  
   PDF exports, invoice sharing, and business branding.

3. **Protect your field records**  
   Signatures, photos, notes, and job history.

4. **Understand the business**  
   Advanced reports.

5. **Future confidence**  
   Cloud backup/sync and team features later.

## 9. Monthly vs Yearly Plan Recommendation

Offer both monthly and yearly.

Recommended structure:

- Monthly: default flexibility
- Yearly: highlighted as best value, about 2 months free
- Do not hide monthly
- Do not preselect yearly in a deceptive way

For tradespeople, yearly can work well because the app supports ongoing business operations, but many solo operators will want to try monthly first.

Suggested presentation:

```text
Monthly
Flexible, cancel anytime

Yearly
Best value - save 20%
```

## 10. Trial Recommendation

Use a **7-day free trial** for Pro.

Why:

- Long enough to create quotes, invoices, and use the app on real jobs
- Short enough to encourage a decision
- Simple for pre-launch testing

Better trigger: offer the trial after activation, not immediately.

Suggested rule:

- Do not show trial on first launch
- Show trial after first completed job, first quote, or first invoice
- Also show it at a limit

## 11. Lifetime Plan Recommendation

A lifetime plan can be useful at launch, but keep it controlled.

Recommended:

- Offer lifetime only as an early adopter plan
- Make it available for the first 100-500 users or during launch month
- Price it high enough to avoid hurting subscription revenue
- Exclude future multi-user/team features if needed

Positioning:

```text
Early Supporter Lifetime
Pay once for ServiceSphere Pro features for solo use.
Future team plans may be separate.
```

Do not make lifetime the main plan forever if cloud sync and ongoing infrastructure costs are coming.

## 13. Soft Paywall Ideas

Soft paywalls should educate without blocking:

- Show locked Pro rows in reports with preview labels
- Let users preview PDF before export
- Allow one sample signature, then ask for Pro
- Show **Pro** badges on advanced features
- Show limit counters, for example `3 of 5 clients used`
- Send gentle in-app reminders after repeated value moments
- Allow users to dismiss upgrade prompts
- Show what they have already created, for example: "You've managed 8 jobs so far"

## 14. Hard Paywall Ideas

Hard paywalls are appropriate for clear premium value:

- Creating a client beyond the free limit
- Creating a job beyond the free limit
- Exporting PDF after free preview/sample
- Removing ServiceSphere branding
- Signature capture after sample
- Advanced reports
- Full photo history beyond free allowance
- Cloud backup/sync when added
- Multi-user/team features when added

Avoid hard-paywalling access to existing records.

## 15. Ethical Paywall Rules For A Field Service App

- Never block access to existing client, job, quote, invoice, photo, or signature records.
- Never surprise-lock data after a trial ends.
- Let users export or view their existing data even if downgraded.
- Make limits visible before users hit them.
- Keep **Continue with Free** available.
- Do not interrupt urgent field workflows.
- Do not use guilt-based copy.
- Clearly explain what happens after trial expiration.
- Restore purchases reliably.
- Make cancellation and subscription management easy through Google Play.

## 16. How To Avoid Hurting Activation

Activation should happen before monetization.

Recommended activation path:

1. Create first client
2. Create first job
3. Add notes/photos
4. Create quote or invoice
5. View/share/export result
6. Then introduce Pro

Do not show a full-screen paywall until the user has done at least one of:

- Created 2 clients
- Created 2 jobs
- Created 1 quote
- Created 1 invoice
- Tried a premium feature intentionally

## 17. How To Test Paywall Copy After Launch

Track:

- Paywall impressions
- Upgrade button taps
- Trial starts
- Purchases
- Trial-to-paid conversion
- Dismissals
- Refunds
- Churn
- Feature usage before paywall

A/B test:

- Headline: "Run more jobs without limits" vs "Upgrade to ServiceSphere Pro"
- Benefit order: unlimited work vs professional PDFs
- Trial CTA: "Start 7-day free trial" vs "Try Pro free"
- Trigger timing: after first invoice vs after limit reached
- Limit size: 3 vs 5 clients, 5 vs 10 jobs
- Annual discount messaging: "Save 20%" vs "2 months free"

Do not optimize only for trial starts. Optimize for retained paid users.

## 18. RevenueCat / Google Play Billing Implementation Notes

Recommended setup:

- Use one entitlement: `pro`
- Use one subscription product with monthly and yearly base plans, or equivalent RevenueCat products grouped into one Offering
- Use RevenueCat Offerings for remote paywall configuration
- Check entitlement status on app launch and before premium actions
- Cache entitlement locally for offline friendliness, but refresh when possible
- Use Google Play test tracks before production
- Support restore purchases
- Handle grace period, billing retry, cancellation, expiration, and refund states
- Keep feature gates server/config driven where possible
- Log paywall trigger source, not just paywall views

## 19. QA Checklist For Paywall

- Free limits trigger correctly
- Existing records remain accessible after limit/trial expiration
- Premium features unlock immediately after purchase
- Restore purchases works
- Cancelled subscription downgrades correctly
- Expired trial downgrades correctly
- Billing retry/grace period is handled
- Offline users do not lose access incorrectly
- Paywall close button works
- Prices match Google Play localized prices
- Monthly/yearly plans display correctly
- Trial terms are clear
- No paywall appears during onboarding
- No paywall blocks saving active work
- Analytics events fire with trigger source
- Test purchases work in Google Play test track

## 20. Final Recommended Paywall Strategy

Use a generous but bounded freemium model: let users manage a small real workload for free, then ask them to upgrade when they hit natural business growth points.

Recommended starting model:

```text
Free:
5 clients
10 jobs
3 quotes
3 invoices
Basic notes/photos/history
Basic dashboard

Pro:
Unlimited clients, jobs, quotes, invoices
PDF exports
Invoice/quote sharing
Signatures
Photo history
Business branding
Advanced reports
Future cloud backup/sync
```

Primary paywall triggers:

1. Hitting client/job/quote/invoice limits
2. Exporting professional PDFs
3. Capturing signatures
4. Adding business branding
5. Opening advanced reports

Main positioning:

```text
ServiceSphere Pro helps you run a more organized, professional service business without limits.
```

This keeps the app fair: free users can experience the workflow and trust the product, while paid users upgrade because ServiceSphere is becoming part of how they actually run their business.
